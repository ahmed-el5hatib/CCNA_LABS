The Scenario: "Pro-Dev Solutions" Office Expansion

"Pro-Dev Solutions" is a fast-growing tech company. Their office is expanding from the "Main Floor" to a new "East Wing" in the same building. The core of the company is the Development team, which needs access to the same secure servers and repositories.

Currently, the Main Floor (managed by SW1) houses the entire Development team and the Administration team.

The expansion plan is to move half of the Development team to the new East Wing (managed by SW2) to make space. The East Wing will also house the new Marketing team.

The Problem: The company requires strict network security. The Admin and Marketing teams must be completely isolated from each other and from the Development network. However, the Development team members must be ableto communicate seamlessly, as if they are in the same room, regardless of whether they are on the Main Floor or in the East Wing.

Scenario Analysis & Requirements

Network & Security Tasks

  • Isolate the three departments (Dev, Admin, Marketing) into separate Broadcast Domains.
  • Extend the Development VLAN across both switches.
  • Ensure Admin and Marketing traffic cannot cross into other VLANs.
  • Implement a single link between switches to carry all necessary traffic (Trunking).

Configuration & CLI Walkthrough

Step 1: Configure SW1 (Main Floor)

A. Create VLANs

enable
configure terminal
!
vlan 10
  name DEVELOPERS
exit
!
vlan 20
  name ADMIN
exit
!
vlan 30
  name MARKETING
exit
Why? We must create all VLANs (10, 20, 30) on the switch. Even though SW1 has no `MARKETING` ports, it must know that VLAN 30 exists so it can allow it to pass over the trunk link (if needed in the future or for routing).

B. Assign Access Ports

! Assign ports to VLAN 10 (Developers)
interface range FastEthernet0/1-10
  switchport mode access
  switchport access vlan 10
exit
!
! Assign ports to VLAN 20 (Admin)
interface range FastEthernet0/11-20
  switchport mode access
  switchport access vlan 20
exit
Why?
  • interface range...: This is a shortcut to configure many ports at once.
  • switchport mode access: This command sets the port to "access" mode, meaning it belongs to one and only one VLAN and will only accept frames that are untagged.
  • switchport access vlan 10: This command assigns the port to VLAN 10. Any device plugged in is now in the "DEVELOPERS" broadcast domain.

C. Configure the Trunk Port

! Configure Gi0/1 as Trunk
interface GigabitEthernet0/1
  switchport mode trunk
end
!
write memory
Why?
  • interface GigabitEthernet0/1: We select the uplink port that connects to SW2.
  • switchport mode trunk: This is the key command. It tells the port to:
    1. Expect frames from multiple VLANs.
    2. Tag outgoing frames with their VLAN ID (using 802.1Q).
    By default, a trunk port allows *all* known VLANs to pass.

Step 2: Configure SW2 (East Wing)

The process is very similar. We must define all VLANs, assign the access ports for *this* switch, and set the other end of the link to "trunk".

A. Create VLANs

enable
configure terminal
!
vlan 10
  name DEVELOPERS
exit
!
vlan 20
  name ADMIN
exit
!
vlan 30
  name MARKETING
exit
Why? Both switches must have an identical VLAN database to understand the tags. If SW2 receives a frame tagged for VLAN 20 (e.g., from a router) but doesn't know VLAN 20, it will drop the frame.

B. Assign Access Ports

! Assign ports to VLAN 10 (Developers)
interface range FastEthernet0/1-10
  switchport mode access
  switchport access vlan 10
exit
!
! Assign ports to VLAN 30 (Marketing)
interface range FastEthernet0/11-20
  switchport mode access
  switchport access vlan 30
exit

C. Configure the Trunk Port

! Configure Gi0/2 as Trunk
interface GigabitEthernet0/2
  switchport mode trunk
end
!
write memory
Why? The link must be a trunk on both sides to negotiate the connection and pass the tagged traffic.

Step 3: Verification & Testing

A. Check VLAN Assignments

On both switches, run:

show vlan brief
What to look for: Confirm that your `Fa0/1-10` ports are listed next to `VLAN 10`, `Fa0/11-20` (on SW1) are next to `VLAN 20`, etc. The trunk port (Gi0/1) will not appear in this list.

B. Check Trunk Status

On both switches, run:

show interfaces trunk
What to look for: This is the proof. You should see your port (`Gi0/1` or `Gi0/2`) with a `Status` of `trunking` and `Encapsulation` of `802.1q`. You should also see `Vlans allowed and active` include 10, 20, and 30.

C. Final Ping Test

(Using the example IPs from our table. These tests confirm both connectivity and isolation.)

  • ✅ Ping from Dev-PC-01 (SW1) to Dev-PC-02 (SW2): SUCCESS (Trunking Works!)
  • ✅ Ping from Dev-PC-01 (SW1) to another Dev PC on SW1: SUCCESS (Intra-VLAN Works)
  • ❌ Ping from Dev-PC-01 (SW1) to Admin-PC-01 (SW1): FAILURE (Isolation Works)
  • ❌ Ping from Dev-PC-02 (SW2) to Mkt-PC-01 (SW2): FAILURE (Isolation Works)
  • ❌ Ping from Admin-PC-01 (SW1) to Mkt-PC-01 (SW2): FAILURE (Isolation across Trunk)
  • ❌ Ping from Admin-PC-01 (SW1) to Dev-PC-02 (SW2): FAILURE (Isolation across Trunk)
  • ❌ Ping from Mkt-PC-01 (SW2) to Admin-PC-01 (SW1): FAILURE (Isolation across Trunk)
  • ❌ Ping from Mkt-PC-01 (SW2) to Dev-PC-01 (SW1): FAILURE (Isolation across Trunk)
